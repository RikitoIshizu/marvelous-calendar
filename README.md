# カレンダー仕様等

## ページ

PC で使う想定のため、SP のデザインは対応していない。
後日追加することを検討中

### カレンダーページ

◯ ページを開くと、当月のカレンダーが表示される。  
○ カレンダーを 1 ヶ月ずつ変更できるボタンがある.  
◯ 当月じゃない場合、カレンダーをリセットできる。(当月の場合はこのボタンを表示しない)  
◯ 数ヶ月以上とばしてカレンダーを見たい場合のために、select タグで年月を選択できる。年は前後 10 年、月は 12 ヶ月を選択することが可能。  
◯ 日付をクリックすると、その日の予定が見れる。

##### 祝日を表示

日付が決まっているものと成人の日や母の日など、週と曜日が決まっていて日付がずれる祝日も該当の日に表示されるようにしている。

##### 祝日ではない特別な日も表示している

クリスマスやハロウィン、日本のひな祭りや節分、元日などの何かしら名称がついている日は表示している。  
ただし、日付が定まっていないものは除く。

### スケジュールページ

その日に登録されているスケジュールとその日が何の日か表示される。
スケジュールの変更、削除が行える。

## 技術選定関連

### バックエンド・DB

#### supabase

-- サーバレスで開発したかった。

-- チートシートもついており、コードの開発の効率性も高い。

ということで、採用。

### 天気予報関連

#### openmeteo

天気予報のデータを取得をするために採用。
無料契約だと、1日1万件までレスポンスを送ることができる。
多くの人に見られることを想定しているアプリケーションではないので、1日1000件を超えることはそうそうない。
リクエストを送ったとしても、数百程度。
そのため、リクエストの条件は満たしている。

また、その他の特徴に関してだと、

-- 取得したい値も豊富で、風速や湿度、降水量、気圧以外に、降雪量等も取得できる
-- 日付指定して取得できるので、1ヶ月分のカレンダーに表示する天気予報のデータを1回で取得可能
-- レスポンスに含まれる天気コードも幅広く、一般的な天気予報の晴れや曇り以外に、雷や霧なども含まれている

と言った点から、本アプリケーションに適切だと判断し、採用。

#### ipstack

軽度と緯度を取得するために使用。
